2.1.0 update

27 April 2026

Overview

On many sites, important UI lives outside the “main” document tree: embedded frames, inert about:blank descendants, or components scoped with shadow DOM. Styles you inject only at the top level often miss those surfaces. This release widens where the same per-host CSS can apply, while staying within what browsers allow an extension to touch.

Bug fixes and reliability

  • Fixed several edge cases around iframe, shadow DOM, popup persistence, and import/export behavior.

2.0.0 update

22 March 2026

Overview

This release changes quite a bit under the hood and in the popup. The goal is fewer surprises: simpler storage, a more predictable editor, and a content script that tracks navigation and applies your CSS more reliably on modern sites.

Prism removed — no syntax highlighting

The Prism-based highlighter is gone. It was tied to a fragile overlay setup and caused real-world bugs (sync with the real field, scrolling, large pastes, focus). There is no syntax coloring anymore: you get a native textarea, line numbers, and the same editing shortcuts as before. The trade-off is clarity and stability over pretty colors.

Local storage only

All site CSS now lives in chrome.storage.local. The old Chrome sync layer and local draft recovery paths are removed so there is a single source of truth per profile. On first launch after this update, existing data in sync is copied into local when a hostname is not already present locally, then removed from sync. Internal draft: keys are cleared.

Popup, content script, and import/export

The popup and content script were rebuilt around straightforward read, debounced save, and live tab preview. Import and export still use the same JSON shape as before. Internally, import/export and storage bookkeeping live in a dedicated script so the main popup file stays easier to follow; the injector keeps one clear path for “what is applied on this tab” to avoid duplicated logic.

Editor and UI

  • Clean white popup with line numbers—no extra editor layer on top of the field.
  • Comfortable for larger stylesheets; UI uses Telegraf for chrome text and a monospace stack for CSS.

Note

CSS is no longer synced across devices by Chrome. Use export/import to move configuration between installations if you need to.

What's New in 1.5.0

8 March 2026

Configuration Import / Export

You can now export your full setup (all sites + CSS + enabled/disabled states) into a JSON file and import it in another browser.

Editor refresh

The editor received a visual refresh focused on CSS editing comfort and a clearer popup layout.

Reliability & Performance Improvements

  • Editor saves are buffered more safely to reduce sync quota issues.
  • Local draft recovery keeps your latest edits if sync is temporarily unavailable.
  • Line number updates are lighter and avoid full redraws in common cases.
  • Unused editor assets were removed to keep the extension lean.

Safer Runtime Behavior

  • Style tag matching/injection is more robust for special hostnames (including IPv6 cases).
  • Import now rejects oversized files before parsing to avoid UI freezes.
  • Popup state reacts better to external storage updates.
